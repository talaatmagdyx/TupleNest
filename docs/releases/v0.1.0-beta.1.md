## TupleNest 0.1.0 — first public beta

**The PostgreSQL IDE that asks before it writes.** Fast, local-first, no
telemetry, no account. Rust + Tauri 2 + React. MIT.

🌐 **Tour with screenshots:** https://talaatmagdyx.github.io/TupleNest/

### Why a beta

The code is tested harder than most v1s — 1,600+ frontend tests, 34 live
PostgreSQL contract tests (13/15/17 in CI), SSH tunnel tests against a real
sshd, and a five-revision adversarial pre-launch review that is public in this
repo. What it has NOT had is users. This release exists to change that, so:

- 🐞 broke something? → [Bug report](../../issues/new?template=bug_report.yml)
- 🤔 nothing broke, it just felt wrong? → [UX feedback](../../issues/new?template=ux_feedback.yml) — this one matters most

### Install

| Platform | File | First run |
| --- | --- | --- |
| macOS (Apple Silicon) | `TupleNest_0.1.0_aarch64.dmg` | not notarized: right-click → Open → Open |
| macOS (Intel) | the Intel `.dmg` | same |
| Windows 10+ | `.exe` or `.msi` | unsigned: More info → Run anyway |
| Linux (glibc 2.35+) | `.AppImage` / `.deb` / `.rpm` | needs webkit2gtk-4.1 + a keyring daemon |

Verify a download (the attestation is the one that means something):

```sh
shasum -a 256 -c SHA256SUMS --ignore-missing
gh attestation verify <file> --repo talaatmagdyx/TupleNest
```

### What's inside

Streaming results grid with honest truncation · schema-aware autocomplete ·
safe row editing (PK-keyed, parameter-bound, refused when racing another
session) · transactions that belong to the tab that opened them · staged
connection tests · TLS verify-full by default, fails closed · host-key-pinned
SSH tunnels · server monitoring with confirm-before-terminate · ER diagram ·
EXPLAIN plans and plan diffing · CSV import · production guard rails enforced
by the server where possible.

Requires PostgreSQL 13+.
