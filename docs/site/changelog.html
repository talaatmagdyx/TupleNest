Current release

v0.1.0-beta.4

Public BetaJuly 2026

Query plans: understanding them, and analysing one you were sent.

Added

  • Analyse a plan you paste — no connection, nothing leaves the machine. TupleNest now reads PostgreSQL's text EXPLAIN output — the indented tree people actually copy out of psql, a ticket, or a colleague's message — as well as FORMAT JSON. It detects which as you type and runs the same analysis it runs on its own queries. The reason to build it is privacy, not convenience: the usual way to get a plan analysed is to paste it into a website, which hands a third party your table names, index names, filter conditions and sometimes literal values out of production. Open it from the activity rail or the command palette — deliberately available while disconnected, since the point is a server this app cannot reach.
  • The text parser was validated against the server itself: ten queries captured as both FORMAT TEXT and FORMAT JSON from a live PostgreSQL 18 — parallel scan, external-merge disk sort, hash spill, a nested loop repeating 20,000 times, a never-executed branch, a CTE, a SubPlan/InitPlan, a trigger, a discarding filter, and a plain scan — then both parsed and compared. All ten agree on node count, depth, loops, rows, estimates, rows removed, blocks, temp blocks, flags, statistics and insights.
  • Richer EXPLAIN ANALYZE plan view. The plan highlights the real bottleneck by self-time — a node's wall time minus its children's — rather than only the costliest sequential scan. A nested loop's inner side really does run once per outer row, so its loops are multiplied; but a node under a Gather reports one “loop” per parallel worker and those run at the same time, so workers are divided back out and the numbers add up to the execution time.
  • Nodes are badged for rows read and then discarded by a filter, sorts that spilled to disk, hashes that spilled to multiple batches, blocks served from disk rather than cache (attributed to the node that actually read them, since PostgreSQL reports buffers cumulatively), estimates that missed by 10× or more, very high loop counts, and branches the executor never reached.
  • Time no node accounts for is surfaced: JIT compilation and trigger time both sit outside the plan tree, and a statement can spend most of its wall clock there. An ordered list of insights turns all of it into next steps.

Fixed

  • Hostnames, role names and key paths are no longer rewritten by macOS. With “capitalize words automatically” on, a typed postgres became Postgres the moment the field lost focus. PostgreSQL role names are case-sensitive, so the connection failed at authentication and looked exactly like a wrong password — with nothing on screen to say the text had been edited after it was typed.
  • The statistics panel says why it is empty. Choosing a non-JSON EXPLAIN format left a bare “—” where an explanation belonged.

Changed

  • The status bar identifies the build it is running — commit and build time, with a trailing + when built from a tree with uncommitted changes. An installed release and a local build are otherwise identical on screen, and telling them apart had already cost an afternoon of debugging a fix inside a window that could not have contained it.
  • CONTRIBUTING.md gains a “Building a real .app locally” section: tauri dev serves the UI from a dev server and goes blank when that server stops, and tauri build refuses without a signing key because releases always ship updater artifacts.

v0.1.0-beta.3

BetaJuly 2026

The security-review response: the release blockers (TLS, numeric DoS, CSV injection), plus data/local/desktop hardening.

Security

  • TLS is now actually required in the verify modes (CRITICAL). The client never set the SSL mode, defaulting to prefer; verify-ca/verify-full now map to require and refuse plaintext — proven against a real ssl=off server with a negative control.
  • A malicious server can no longer OOM the app via a numeric value (HIGH). The wire display scale was read signed and cast to a huge loop bound; now read unsigned and clamped, with digit/render ceilings.
  • CSV export neutralizes spreadsheet formula injection (HIGH). Cells starting with = + - @ (or tab/CR) are neutralized by default; a toggle allows raw output.
  • On-disk stores, logs, and crash reports are now owner-only on Unix (0700 dir / 0600 files).
  • Stored SQL is scrubbed of secret literals; -- tuplenest:no-history skips a statement's history row.
  • A single huge cell can no longer spike memory — rendered values capped at 1 MiB with a truncation marker.
  • Keychain entries are no longer orphaned on every Test/connect — a held reference is reused.
  • Exports go through a Rust command that owns the save dialog; the fs plugin was removed. CSP tightened (object-src/base-uri/frame-ancestors none).
  • Auto-update anti-rollback floor; release logging caps chatty crates to warn.
  • All third-party GitHub Actions pinned to commit SHAs; CI runs least-privilege.
  • Failed-query history no longer persists row values — only title, SQLSTATE, and category.
  • Deterministic fuzz (300k+ inputs) over the numeric and raw-value decoders.

Fixed

  • A failed query now shows the server's full report (SQLSTATE, message, DETAIL, HINT, CONTEXT), not the bare “Database error”.

v0.1.0-beta.2

BetaJuly 2026

One real bug, found by the first macOS tester within an hour of beta.1 — which is exactly what betas are for.

Fixed

  • The window close button did nothing. The close guard ended in destroy(), but the capability file granted only core:default, which doesn't include allow-destroy. One granted permission was the whole fix.
  • The same gap silently broke the open-transaction close prompt — “Commit & quit” / “Rollback & quit” could never actually quit. Both are now verified against a live database.

Added

  • A test that scans the frontend for every getCurrentWindow() call and asserts the capability grants it — verified by removing the permission and watching it fail.

Docs

  • README rewritten: sixty-second quick start, real keyboard map, honest “what TupleNest is not (yet)”.
  • Corrected what a Windows code-signing certificate actually buys (OV doesn't skip SmartScreen; only EV does).

v0.1.0-beta.1

BetaJuly 2026

First public release — to put it in front of real users, because “is the UX intuitive” is a question no test suite can answer.

Fixed (pre-launch review)

  • The open transaction now has an owning tab; committing from another tab is refused and names the owner.
  • Row edits check rowsAffected; a statement must touch exactly one row or the set rolls back.
  • The destructive-statement guard masks comments/strings first — a commented-out WHERE no longer disarms it.
  • Read-only profiles issue a real read-only session characteristic instead of being hard-coded off.
  • Terminating a backend asks first; generated/identity columns are read-only; the grid footer counts the real total.

Added

  • SECURITY.md, PRIVACY.md, the changelog.
  • CI runs contract tests against real PostgreSQL 13, 15, and 17 with TLS enabled.
  • Read-only connection profiles.

Known limitations

  • macOS builds not notarized; Windows installers unsigned — first launch needs one extra click.
  • CSV import reads the whole file into memory.
  • Nobody has lived with this yet. That's what the beta is for.
