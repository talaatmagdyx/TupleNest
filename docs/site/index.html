PRODUCTION SAFETY

Production safety is not a warning color. It is a workflow.

Connections are classified as development, staging, or production, and production connections are visually distinct — you always know where a query is about to run.

  • Read-only mode issues a real PostgreSQL read-only session characteristic
  • Destructive statements require explicit confirmation
  • The open transaction is owned by a tab, so you can't commit another tab's work
TupleNest reduces accidental risk. A database role without write privileges remains the strongest security boundary — the app is a guardrail, not the final authority.
SQL EDITOR

An editor that understands PostgreSQL.

Schema-aware completion for tables, columns, and functions; PostgreSQL-correct literal masking (dollar-quoting, nested comments) so the destructive-statement guard reads the real query, not raw text.

  • Formatting & selected-query execution
  • Multiple tabs with unsaved-change indicators
  • Query history & snippets

RESULTS & PERFORMANCE

Large results without losing control.

Rows stream from a bounded store and render through a virtualized grid, so a wide or long result stays responsive instead of freezing the window.

  • Backend row and byte budgets, with clear truncation indicators
  • Per-cell size limit so one giant value can't spike memory
  • Query cancellation, execution duration, and true row count
The grid caps rows and bytes — the footer counts the real total, not the cap. We don't publish throughput numbers we haven't benchmarked.
SCHEMA & RELATIONSHIPS

  • Global object search
Security

Your data stays yours.

Security is the foundation, not a feature bolt-on. Here's the short version — the full model is on the Security page.

Local-first

No telemetry

OS keychain

TLS verification

SSH host verification
