Safety is the product

Every guarantee below is enforced in a specific place — by the server where possible, by reviewed SQL where not — and every one has a test that performs the misuse and asserts the refusal.

GuaranteeHow it's actually enforced
Read-only profilesPostgreSQL itself: SET SESSION CHARACTERISTICS … READ ONLY. The server refuses writes — not a client-side promise.
Destructive guardUPDATE/DELETE without WHERE, plus DROP / TRUNCATE / ALTER / GRANT…, ask before running on prod and staging. SQL is masked first — comments, strings, dollar-quotes — so a commented-out WHERE can't disarm it. Thirteen adversarial bypasses are pinned by tests. Best-effort by design: the seatbelt light, not the seatbelt.
Safe row editingOnly single-table SELECTs carrying the full primary key are editable; joins, groups, CTEs and derived tables are refused with a reason. You review the exact generated UPDATEs before anything runs. Writes are parameter-bound, keyed by the full PK, and guarded with IS NOT DISTINCT FROM on old values — an edit racing another session hits 0 rows and is refused, not silently won.
Transactions own their tabOne session serves all tabs, so the tab that opened a transaction owns it. Commit from another tab is refused and names the owner. Closing the window mid-transaction always prompts.
CredentialsOS keychain only — Keychain / Credential Manager / Secret Service. Never in config files, state, logs, or IPC payloads; the backend resolves opaque references.
TLS fails closedverify-full by default; a bad chain, expired cert or wrong name is an error, never a silent downgrade. verify-ca is real and distinct — chain checked, name knowingly skipped — for SSH tunnels where you dial localhost.
SSH tunnels pin host keysFingerprint or known_hosts; an unknown or wrong host key fails closed. Tested against a live sshd in CI.
Signed auto-updatesVerified against a minisign public key compiled into the binary. The release host serves bytes; it never holds the private key. Updating is refused while a transaction is open.

A tour of the real thing

Screenshots below are unretouched captures of v0.1.0 against a local PostgreSQL 18.

Connections that prove themselves

A staged probe — DNS, TCP, auth, server version — tells you which layer failed instead of "could not connect". Environments (dev / test / staging / prod) change the window's ambient color and arm the production guards.

The password field says where the password goes, because a promise you can't see is a promise you can't check.

Autocomplete that read your schema

Tables after FROM, columns of the in-scope tables in WHERE, alias. resolving to that table's columns, schema. to its tables. Comments and string literals are masked before context is computed, so nothing inside them ever poisons a suggestion.

Plus: Format SQL, EXPLAIN / EXPLAIN ANALYZE plans, $1..$n parameter binding, searchable history, snippets in the command palette.

A grid that streams and tells the truth

Virtualized and backpressured: rows arrive in bounded batches, the first 100,000 are kept under both a row cap and a byte budget, and the footer reports truncation honestly. A checksum test pins that nothing is lost or duplicated in transit.

Unknown types are decoded from what the type is, never guessed from what the bytes look like — money renders as money or as visibly-raw hex, never as a plausible wrong number. Full keyboard navigation, real role="grid" semantics.

The schema, as a picture

The ER view draws the foreign-key graph and names every constraint. Alongside it: schema diff (columns, types, nullability, PKs), find-usages and rename across tabs, plan-vs-plan comparison with a "new sequential scan" regression flag, and partition tree browsing.

Operate, not just query

Live backends, cache hit ratio, commits and rollbacks, blocked locks, and per-session detail — with cancel and terminate. Terminate always confirms; on production it makes you type the pid.

Health panels for index usage, vacuum & bloat, and pg_stat_statements top queries round it out.

CSV import wizard

RFC-4180 parsing — quoted delimiters, escaped quotes, embedded newlines, BOM. You review inferred names and types before anything runs; inserts are batched inside one transaction. Whole file is read into memory, so huge CSVs aren't practical yet.

Production audit log

Everything executed on a prod-tagged connection is kept, full SQL, in a dedicated audit log — while regular history omits prod SQL for privacy. Accountability and privacy, separately.

Global object search

One box over every schema, table, view, column, index and function the cache knows, with instant results served locally.

Metadata cache

Explorer and completion answer from a local SQLite cache that refreshes live and serves offline — a schema with 13,000 relations opens as a tree, not a spinner.

Crash logs stay local

No telemetry, no account, no cloud. Panic reports are written to local disk with key/value and URL-credential redaction, and are never uploaded.

Accessible by construction

Real dialog semantics with focus traps on all 28 modals, roving-tabindex grid keyboard model, prefers-reduced-motion respected.

Free, MIT-licensed, built from source in public CI. PostgreSQL 13+ (13, 15 and 17 are contract-tested on every push).

Gatekeeper is a funding problem: an Apple account and notarization make it stop. SmartScreen is not the same, and this page used to imply it was — a certificate puts a real name on the installer, but the warning stays until the download earns reputation, and that resets with every release. Neither dialog tells you what you actually downloaded. The checksums below do.

Verify what you downloaded

Two checks, honestly labeled — because they prove different things.

# catches a corrupted or truncated download
shasum -a 256 -c SHA256SUMS --ignore-missing

# binds the file to the CI build that produced it — signed by GitHub's
# OIDC identity, not by anything sitting in the release. The one worth running.
gh attestation verify TupleNest_0.1.0_aarch64.dmg --repo talaatmagdyx/TupleNest

SHA256SUMS sits beside the installers and is unsigned — whoever could swap an installer could swap it too. The attestation can't be swapped. CycloneDX SBOMs (one per Rust crate, one for the npm tree) ship with every release. Auto-updates verify a minisign signature against a key compiled into the binary and need nothing from you.
