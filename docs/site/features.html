SQL EDITOR

Smart SQL editor

Built for PostgreSQL specifically — the literal masking understands dollar-quoting, nested comments, and E'' escapes, so features like the destructive-statement guard read the real query rather than raw text.

  • PostgreSQL syntax highlighting
  • Schema-aware completion: tables, columns, functions, keywords
  • Formatting and selected-query execution
  • Multiple editor tabs with unsaved-change indicators
  • Line and block comments, find and replace
  • Query history and reusable snippets
  • Query parameters ($1..$n) with binding
RESULT GRID

Results that stay responsive

Rows stream from a bounded store into a virtualized grid, so large or wide results don't freeze the window — and the footer counts the real total, not the cap.

  • Streaming result loading, virtualized rows
  • Column resizing, sorting, filtering, search
  • Explicit NULL representation
  • Copy cell, row, or selection
  • Export (CSV / JSON / Markdown)
  • Query duration, row count, truncation indicators
Query performance

See why a query is slow

EXPLAIN-based tooling to inspect plans and compare them, plus schema-level diffing for change review.

Available
  • EXPLAIN and EXPLAIN ANALYZE
  • Text plan inspection
  • Plan comparison between runs
  • Planning vs execution time
  • Schema diff for change review
Planned
  • Richer visual plan with expensive-node highlighting
  • Sequential-scan & sort/join call-outs
  • BUFFERS-aware view
  • A dedicated query profiler
Where a richer visualization isn't built yet, it's listed here as planned rather than shown as done. See the Roadmap.
SCHEMA EXPLORER

Browse the whole catalog

A lazy tree that loads schemas, objects, and columns on demand and serves from a local cache when you're offline.

  • Databases, schemas, tables, views, materialized views
  • Functions, enums, indexes, constraints, triggers
  • Foreign keys and dependencies
  • Global object search and refresh
  • Open an object straight into the editor
ER DIAGRAMS

The database as a graph

An automatic relationship graph built from real foreign keys — pan, zoom, focus a table, and expand its neighbours.

  • Table cards with foreign-key links
  • Pan & zoom, focus on a selected table
  • Search and expand related tables
  • Open table data or definition
PRODUCTION SAFETY

Safety as a workflow

Classify each connection, make production visually distinct, and let PostgreSQL itself enforce read-only when you ask for it.

  • Environment classification and a production badge
  • Read-only session characteristic (server-enforced)
  • Destructive-statement detection and confirmation dialogs
  • Transaction-state visibility and tab ownership
  • Query timeout controls and clear warning language
  • Least-privilege guidance

Import & export

Move data in and out, safely

Import

  • CSV import with header mapping
  • Type & error preview before you commit
  • Transaction mode with cancel support

Export

  • CSV, JSON, Markdown, clipboard
  • Spreadsheet-safe CSV on by default (formula-injection neutralized)
  • Raw-value mode when you need it; column selection
Cells starting with = + - @ (or tab/CR) are neutralized so Excel, Sheets, and LibreOffice treat them as text, not formulas. A toggle lets you opt into raw output.
