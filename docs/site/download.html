Verify your download

Trust, then verify

Because the installers aren't OS-signed, the published signature and checksum are the real integrity signal. Both take one command.

SHA-256

# compare against the SHA256SUMS file in the release
shasum -a 256 <filename>
# or on Linux
sha256sum <filename>

minisign

# verify the signature with the published public key
minisign -Vm <artifact> -P <public-key>
Each release also carries a SHA256SUMS file, per-artifact signatures, the minisign public key, an SBOM (CycloneDX), and a GitHub build-provenance attestation you can check with gh attestation verify.
Installation

Install & first launch

🍎 macOS

  • Open the .dmg, drag TupleNest to Applications
  • First launch: right-click the app → Open, then confirm
  • Gatekeeper only warns once per version
  • Uninstall: move the app to Trash

🪟 Windows

  • Run the .msi or .exe installer
  • SmartScreen: More info → Run anyway
  • Requires WebView2 (bundled on Win 11, auto-installed otherwise)
  • Uninstall: Apps & features

🐧 Linux

  • AppImage: chmod +x then run
  • .deb: sudo dpkg -i tuplenest_*.deb
  • Needs webkit2gtk installed
  • Uninstall: sudo apt remove tuplenest
Build from source

Prefer to build it yourself?

The whole thing compiles from a clean checkout. You'll need Rust (stable), Node 22, npm, and the Tauri prerequisites for your OS.

# clone
git clone https://github.com/talaatmagdyx/TupleNest
cd TupleNest/apps/desktop

# install frontend deps
npm install

# run in development
npm run tauri dev

# production build (installers land in src-tauri/target/release/bundle)
npm run tauri build
Tauri prerequisites differ per platform (Xcode CLT on macOS; MSVC + WebView2 on Windows; webkit2gtk + build-essential on Linux). See the Tauri docs linked from the repository README.

The app refuses an update advertised below a compiled-in minimum version (anti-rollback). During the beta, updates are delivered by downloading the newer build; the auto-updater targets stable “latest” releases only.
