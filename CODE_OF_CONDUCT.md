# Code of conduct

## The short version

Be decent. Assume good faith. Criticise code, not people.

This project's whole thesis is that a claim should be checkable and a mistake
should be admitted plainly — the repository's own review document is a list of
things it got wrong. That only works if being wrong is safe here. So: no
sneering at a beginner's question, no point-scoring in a review, no treating a
bug report as an accusation.

## Specifically

**Welcome:** questions that seem obvious, bug reports without a reproduction,
"this felt wrong and I can't say why", disagreement with a maintainer,
pointing out that the docs lie.

**Not welcome:** harassment or demeaning comments about anyone's identity —
age, body, disability, ethnicity, sex, gender identity or expression, level of
experience, nationality, appearance, race, religion; sexualised language or
imagery; personal or political attacks; publishing others' private information;
sustained disruption; deliberately posting someone else's database credentials
or private schema in an issue.

**Applies to:** issues, pull requests, discussions, commit messages, code
comments, and anywhere someone is representing the project.

## Reporting

Email **talaatmagdy75@gmail.com**. Reports are read by the maintainer and kept
confidential. If a report concerns the maintainer, say so and it will be
handled by an independent third party — this project is one person today, and
pretending otherwise would be exactly the kind of comfortable fiction it tries
to avoid.

You will get an acknowledgement within a few days. If a response is warranted
it will be, in escalating order: a private word, a public correction, removal
of a comment, or a ban. Whichever it is, you will be told what happened and
why.
